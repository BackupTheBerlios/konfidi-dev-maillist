I think we really need to use exponentiation to slow the falloff rate in
the multiplicative strategy.  Take our demo for example.  Say I trust
Alice .8, she trusts Bob .7, and he trusts Clara .9.  Reasonable, right?
  But we get:

.8 * .7 * .9 = .504

This is way too low.  I would imagine .6 or .7 would be the lowest
cutoff most clients would have.

Andy, can you implement this easily?  An exponent of 1/2 seems to be
reasonable, but it should be configurable.

.8^(1/2) * .7^(1/2) * .9^(1/2) =
.894 * .837 * .949 = .710

WDYT?  We should write about this too

Other examples with a 1/2 exponent seem reasonable too:
.8 # .7 # .9 = .71
.5 # .5 = .5
.4 # .3 = .346
.3 # .7 = .458
